SX - How it works

SX Cluster is a reliable, fully distributed cluster solution for your data storage needs, released under the GPL licence. With SX Cluster you aggregate the disk space available on multiple servers and merge it into a single storage system. The cluster makes sure that your data is always replicated over multiple nodes (the exact number of copies is defined by the sysadmin) and synchronized. Additionally, SX Cluster has built-in support for high-availability, bit rot detection, deduplication, client-side encryption, on-the-fly compression and much more.

Support SXFS

Features

Deduplication

If your users upload 10 copies of the same data, that data takes the same space as 1 copy. If you upload 10 files which differ only for a few bytes, only the differences between the files take up additional space, the rest is deduplicated.

Fast and lightweight protocol

Never transfer the same data twice. The SX protocol transfers only the differences between the local copy of a file and the data already stored in the cluster. Additionally, it transfers data to/from all SX nodes in parallel to maximize speed.

Indexing and search

Need to find that important document quickly? With our SXIndexer commercial plugin for ElasticSearch you can index and search the content of docs, pdfs, spreadsheets and slides stored in your cluster.

Revisions and undelete

SX Server can optionally keep multiple revisions of your files and allow you to go back in time to any version of your files. You can also restore a file that has been accidentally deleted.

Replication

Choose how many times you want your data to be replicated. You can set different levels of replica for different data and find the perfect balance between reliability and efficiency.

Encryption

Your data is safe by default. Server-to-server and client-to-server communications are securely encrypted by default. Additionally SX supports client-side encryption, which ensures that your data cannot be read by the sysadmin of the cluster.

Rack awareness

By default, data is distributed evenly across all the nodes in the cluster: if you want to achieve redundancy across two or more racks of servers, or across two or more datacenter, partition your cluster in multiple zones.
